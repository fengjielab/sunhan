# D435i 到 Panda 机械臂手眼标定指南

这份文档按“从零开始能真正做出来”为目标整理，默认你要做的是**眼在手上（eye-in-hand）**标定，也就是把 Intel RealSense D435i 固定在 Panda 机械臂末端，求出`末端执行器 -> 相机`的刚体变换。

如果你的相机不是装在手上，而是固定在工作台外部，请看本文末尾的“眼在手外（eye-to-hand）”说明；本文附带的脚本也支持这两种模式。

## 1. 你最终要得到什么

手眼标定的最终结果通常是一个 4x4 齐次变换矩阵：

`T_gripper_camera`

它表示：已知 Panda 末端执行器坐标系中的点，如何变换到 D435i 相机坐标系；或者反过来通过求逆得到`T_camera_gripper`。

如果后续你要做抓取、视觉伺服或相机点云投到机器人基座，常用的是：

`T_base_camera = T_base_gripper * T_gripper_camera`

所以整个标定链条里，至少要保证下面两类数据是可靠的：

1. 机器人每次采样时的末端位姿 `T_base_gripper`
2. 标定板相对相机的位姿 `T_camera_target`

## 2. 推荐硬件和准备

推荐准备如下：

1. `Intel RealSense D435i`
2. `Franka Emika Panda`
3. 一块刚性足够好的标定板
4. 一个稳定、不晃动的相机支架
5. 一台能同时读取相机和机器人位姿的电脑

标定板推荐优先级：

1. `ChArUco` 板：比纯棋盘格更稳，遮挡时也更容易识别
2. `AprilTag` 板：也很好用，尤其在姿态变化较大时
3. 纯棋盘格：能做，但对角点质量和视角要求更高

对 D435i 的建议：

1. 手眼标定优先用 **RGB 彩色相机**，不要直接拿深度图做位姿解算
2. 标定过程中分辨率、帧率、曝光参数尽量固定
3. 相机支架必须足够硬，采样过程中不能有松动
4. 如果相机装在夹爪附近，确认整条运动范围内都不会碰到标定板

## 3. 坐标系约定

先统一约定，后面不容易乱：

1. `base`：Panda 机器人基座坐标系
2. `gripper`：Panda 末端执行器坐标系
3. `camera`：D435i 彩色相机坐标系
4. `target`：标定板坐标系

本文使用下面这个记号：

`T_A_B` 表示“把 B 坐标系中的点变换到 A 坐标系”。

例如：

1. `T_base_gripper`：夹爪在机器人基座下的位姿
2. `T_camera_target`：标定板在相机坐标系下的位姿
3. `T_gripper_camera`：相机在夹爪坐标系下的位姿

## 4. 从零开始的完整流程

### 4.1 固定相机

把 D435i 刚性固定到 Panda 末端，注意：

1. 安装完成后，用手轻推支架，确认没有明显弹性或回弹
2. 线缆要做好应力释放，不能随着关节运动拉扯相机
3. 安装完成后，在整个标定期间不要再拧动支架

如果这一步不稳，后面的标定结果通常会漂。

### 4.2 准备标定板

有两种常见摆法：

1. `eye-in-hand`：标定板固定在桌面或支架上，相机跟着机械臂运动
2. `eye-to-hand`：相机固定在外部，标定板装在机械臂末端

对于你现在“D435i 装在 Panda 上”的情况，推荐第一种：

把标定板固定在工作空间里一个稳定位置，不要手拿。

建议：

1. 标定板要足够平整
2. 尺寸不要太小，建议至少 A4 左右
3. 固定位置要让相机能从多个角度看到它

### 4.3 先做相机内参标定或确认内参

手眼标定依赖你每一帧里对标定板姿态的估计，所以相机内参必须先靠谱。

你有两条路：

1. 直接使用 D435i 当前分辨率下的 RGB 内参
2. 自己重新做一次更严格的 RGB 内参标定

推荐做法：

1. 选定一个最终使用的 RGB 分辨率，例如 `1280x720` 或 `1920x1080`
2. 标定期间始终保持这个分辨率不变
3. 保存对应的相机矩阵 `K` 和畸变参数 `D`

注意：

1. 手眼标定和后续运行时，必须使用同一颗相机、同一分辨率、同一对齐设置
2. 如果后面把彩色图对齐到深度图，或者切换了分辨率，内参就可能变了

### 4.4 确认 Panda 输出的是哪个末端坐标系

这一点非常重要。

你需要确认从 Panda 读出来的位姿到底是：

1. 法兰盘 `flange`
2. 工具中心点 `TCP`
3. 夹爪某个自定义坐标系

手眼标定里的 `gripper` 必须和你后续控制、抓取时使用的末端坐标系保持一致。

推荐做法：

1. 先选定一个末端参考系，比如 `panda_hand` 或你自定义的 TCP
2. 采样时始终记录这个坐标系相对 `base` 的位姿
3. 后续所有使用标定结果的地方，也统一基于这个参考系

### 4.5 采集标定样本

这是最关键的一步。你需要采集多组同步样本，每组样本包含：

1. `T_base_gripper`
2. `T_camera_target`

一组样本对应机械臂的一个静止姿态。

推荐采样数量：

1. 最少 `15` 组
2. 实用上建议 `20` 到 `30` 组
3. 如果姿态分布不够丰富，拍再多也没用

姿态采样原则：

1. 不要只在一个平面上移动
2. 要有明显的平移变化
3. 更要有明显的旋转变化
4. 让标定板出现在图像中心、边缘、近处、远处
5. 包含俯视、侧视、倾斜视角

经验上：

1. 单次采样前先让机械臂完全静止
2. 采图和读机器人位姿尽量同步
3. 机械臂不要在运动过程中抓拍
4. 每一帧都检查标定板检测质量，不好的帧直接丢弃

### 4.6 每一帧怎么得到 `T_camera_target`

如果你用的是 ChArUco 或棋盘格，典型流程是：

1. 在 RGB 图像中检测角点
2. 用相机内参 `K, D` 做 PnP
3. 解出标定板相对相机的位姿

这个位姿通常就是：

`T_camera_target`

也就是“target 在 camera 下的位姿”。

如果你用 OpenCV，最后拿到的通常是 `rvec, tvec`，它们对应的就是目标相对于相机的位姿。

### 4.7 每一帧怎么得到 `T_base_gripper`

从 Panda 控制接口里读取当前末端位姿。

常见做法：

1. 如果你在 ROS 中，用 TF 或机器人状态接口读当前末端位姿
2. 如果你直接用 Franka 接口，就读取当前末端齐次矩阵
3. 存盘时统一保存为平移 + 四元数，或者直接 4x4 矩阵

建议：

1. 四元数统一使用 `xyzw`
2. 长度单位统一用 `米`
3. 不要混用毫米和米

## 5. 数据文件格式

本目录里已经放了一个模板文件：

`samples.template.json`

注意：这个模板里的数值只是**占位示例**，用于说明格式，不是有效标定数据，不能直接拿来当真实结果。

每个样本长这样：

```json
{
  "name": "pose_001",
  "gripper_wrt_base": {
    "translation": [0.45, 0.02, 0.31],
    "quaternion_xyzw": [0.0, 0.0, 0.0, 1.0]
  },
  "target_wrt_camera": {
    "translation": [0.03, -0.08, 0.62],
    "quaternion_xyzw": [0.12, -0.03, 0.70, 0.70]
  }
}
```

含义如下：

1. `gripper_wrt_base` 就是 `T_base_gripper`
2. `target_wrt_camera` 就是 `T_camera_target`

## 6. 求解手眼标定

本目录里附带了脚本：

`handeye_calibrate.py`

它使用 OpenCV 的 `calibrateHandEye` 实现求解，支持：

1. `eye_in_hand`
2. `eye_to_hand`

默认命令：

```bash
python3 /home/mfj/biaoding/handeye_calibrate.py \
  --samples /home/mfj/biaoding/samples.template.json \
  --mode eye_in_hand \
  --method all
```

如果你已经把真实采样写到 `samples.json`，就把路径换掉即可。

脚本会输出：

1. 解出的 4x4 变换矩阵
2. 平移向量
3. 四元数
4. 一个简单的一致性误差统计

建议做法：

1. 先跑 `--method all`
2. 比较不同算法的结果是否接近
3. 如果某个算法明显偏离其他算法，优先怀疑样本质量

## 7. 如何判断标定结果好不好

不要只看脚本能不能输出矩阵，要做验证。

### 7.1 数值验证

对于 `eye-in-hand`，理论上：

`T_base_target = T_base_gripper * T_gripper_camera * T_camera_target`

对于所有样本都应该接近同一个常量。

也就是说，如果你把每一帧推算出来的 `T_base_target` 摆在一起，它们应该很接近。

重点看：

1. 平移误差是否稳定在毫米级到厘米内
2. 旋转误差是否稳定
3. 有没有少数明显离群样本

### 7.2 实际验证

推荐做两个落地验证：

1. 把标定板角点投回图像，看重投影是否对得上
2. 选一个空间固定点，比较视觉估计和机器人实际位置是否一致

更直接一点：

1. 让相机识别标定板中心点在 `camera` 下的位置
2. 用 `T_base_camera` 把它变到 `base`
3. 控制 Panda 末端去接近这个点
4. 看是否能稳定到达附近

如果结果波动很大，优先排查：

1. 相机支架是否晃
2. 标定板是否平整
3. 采样姿态是否过于单一
4. 相机内参是否和当前分辨率一致
5. 机器人记录的末端坐标系是否与你后续实际使用的一致

## 8. 采样建议：一套实用动作

你可以按下面这个思路采 20 到 30 组：

1. 正视标定板，距离近一点
2. 正视标定板，距离远一点
3. 左侧斜看
4. 右侧斜看
5. 上方俯看
6. 下方仰看
7. 让标定板落在图像左上区域
8. 让标定板落在图像右上区域
9. 让标定板落在图像左下区域
10. 让标定板落在图像右下区域

要点不是机械臂走得多花，而是：

1. 视角变化大
2. 板子始终能被高质量检测
3. 机器人姿态覆盖充分

## 9. D435i + Panda 常见坑

### 9.1 把深度相机和 RGB 相机坐标系混了

D435i 内部不同传感器之间有外参。手眼标定如果是用 RGB 图像检测标定板，那你解出来的就是**RGB 相机**相关外参，而不是深度坐标系。

如果后面要把深度点云用起来，需要再通过 RealSense 已知外参把深度系转到 RGB 系，或者反过来。

### 9.2 末端坐标系不一致

采样时用的是 `flange`，运行时却用 `TCP`，误差会直接体现在抓取偏差上。

### 9.3 样本姿态太相似

这是最常见的问题之一。很多人采了 30 张，但其实都是“站在一个位置轻微晃动”，这种数据对手眼标定帮助不大。

### 9.4 采样不同步

图像是这一瞬间的，机器人位姿是另一瞬间的，如果机械臂没停稳，会造成明显误差。

### 9.5 支架微小松动

哪怕只动了一两毫米，都会让手眼结果发散。采样前后最好肉眼确认支架没有发生变化。

## 10. 眼在手外（eye-to-hand）怎么改

如果你的 D435i 不是装在 Panda 手上，而是固定在工作台旁边，那么流程改成：

1. 相机固定不动
2. 标定板固定在 Panda 末端
3. 每帧仍然记录机器人末端位姿和标定板相对相机位姿
4. 这时要求的是 `T_base_camera`

本目录脚本可以通过：

```bash
python3 /home/mfj/biaoding/handeye_calibrate.py \
  --samples /home/mfj/biaoding/samples.json \
  --mode eye_to_hand \
  --method all
```

来处理这类数据。

## 11. 推荐你实际执行时的顺序

如果你现在就要开始做，建议按下面顺序：

1. 把 D435i 刚性装到 Panda 末端
2. 选定一个固定的 RGB 分辨率
3. 确认并保存这套分辨率下的内参
4. 明确 Panda 末端参考系到底是哪一个
5. 准备并固定标定板
6. 采 20 到 30 组高质量样本
7. 把样本填进 `samples.json`
8. 跑 `handeye_calibrate.py`
9. 比较多种方法结果
10. 做实际验证

## 12. 你下一步最值得做的事

如果你希望真正开始干，不要先纠结算法，先做这三件事：

1. 确认你是 `eye_in_hand` 还是 `eye_to_hand`
2. 确认 Panda 输出的末端位姿对应哪个坐标系
3. 先采 5 组样本手工检查数据格式是否正确

等这三步没问题，再一次性采满正式数据。

## 13. 标定板制作

我已经把标定板生成脚本放在：

`generate_charuco_board.py`

默认会生成一套适合你当前场景的 A4 `ChArUco` 标定板：

1. `boards/charuco_a4_5x7_30mm.png`
2. `boards/charuco_a4_5x7_30mm.pdf`
3. `boards/charuco_a4_5x7_30mm.json`

生成命令：

```bash
python3 /home/mfj/biaoding/generate_charuco_board.py
```

默认参数：

1. 棋盘格 `5 x 7`
2. 方格边长 `30 mm`
3. ArUco 码边长 `22 mm`
4. 字典 `DICT_5X5_100`
5. 纸张 `A4`

打印时注意：

1. 一定要选择 `100%` 比例打印
2. 不要选“适应页面”
3. 打印后用尺子测一个方格，确认是 `30 mm`
4. 最好把纸贴到硬板上，避免卷曲
