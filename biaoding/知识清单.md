# 手眼标定项目 — 知识清单

以下是整个对话涉及的全部知识点，按领域分类。

---

## 一、坐标系与三维变换（最核心）

| 概念 | 说明 |
|------|------|
| **坐标系** | 基坐标系（world/base）、法兰盘坐标系（gripper/endeffector）、相机坐标系（camera）、标定板坐标系（target） |
| **齐次变换矩阵 $T$** | $4\times4$ 矩阵，表示两个坐标系之间的平移+旋转：$\begin{bmatrix} R & t \\ 0 & 1 \end{bmatrix}$ |
| **旋转矩阵 $R$** | $3\times3$ 正交矩阵，$R^T R = I$，$\det(R)=1$，表示纯旋转 |
| **平移向量 $t$** | $3\times1$ 向量，表示坐标系原点间的位移 |
| **矩阵乘法链** | $T_{\text{base}}^{\text{target}} = T_{\text{base}}^{\text{gripper}} \cdot T_{\text{gripper}}^{\text{camera}} \cdot T_{\text{camera}}^{\text{target}}$ |
| **坐标变换 vs. 点变换** | 变换坐标系 vs. 在同一坐标系下移动一个点 |
| **列主序存储** | libfranka 的 `O_T_EE` 是 16 个元素的列主序矩阵 |

> **🔗 推荐学习**：先看 3Blue1Brown 的线性代数系列（"线性变换"那几集），然后看《视觉 SLAM 十四讲》第 3 讲。

## 二、旋转的表示方法

| 表示法 | 维度 | 优点 | 缺点 | 代码中使用 |
|--------|------|------|------|-----------|
| **旋转矩阵 $R$** | $3\times3$ | 直观、可链乘 | 9 个参数冗余，必须保证正交 | `cv2.Rodrigues()` 转换 |
| **旋转向量（轴角）** | $3\times1$ | 紧凑，OpenCV 默认 | 有奇异性 | `cv2.Rodrigues()`、`cv2.solvePnP` 的 `rvec` |
| **欧拉角（RPY）** | 3 个角 | 直观（俯仰/偏航/横滚） | 有 Gimbal Lock（万向锁） | 本项目未用 |
| **四元数** | $[x,y,z,w]$ | 紧凑、无奇异性、可平滑插值 | 不直观 | `rotation_to_quat()`、Slerp 插值 |

### 四元数（重点补）

- 单位四元数 $\|q\|=1$，表示纯旋转
- 共轭 $q^* = [-x, -y, -z, w]$ 表示反向旋转
- 乘法：两个旋转的复合
- **Slerp**：球面线性插值，用于两个姿态之间平滑过渡
- 从旋转矩阵 → 四元数的转换（代码中的 `rotation_to_quat()`）

> **🔗 推荐学习**：3Blue1Brown "四元数" 视频；《视觉 SLAM 十四讲》第 3 讲关于四元数部分。

## 三、手眼标定

| 概念 | 说明 |
|------|------|
| **问题定义** | 已知 $T_{\text{base}}^{\text{gripper}}$ 和 $T_{\text{camera}}^{\text{target}}$ 的多组对应值，求解固定的 $T_{\text{gripper}}^{\text{camera}}$ |
| **$AX=XB$ 方程** | 手眼标定的核心数学形式：$A \cdot X = X \cdot B$，其中 $A$ 是两次末端位姿的变换，$B$ 是两次相机位姿的变换 |
| **眼在手上（eye-in-hand）** | 相机固定在机器人末端，标定板固定在世界中 |
| **眼在手外（eye-to-hand）** | 相机固定在世界中，标定板在机器人末端 |
| **Tsai 方法** | 两步法：先解旋转再解平移 |
| **Park 方法** | 基于李群 $SE(3)$ 的优化方法 |
| **Horaud 方法** | 基于四元数的解法 |
| **Andreff / Daniilidis 方法** | 使用旋量理论或对偶四元数 |
| **一致性评估** | 用标定结果把所有样本投影到基坐标系，看平移和旋转的方差 |

> **🔗 推荐学习**：搜索 "Hand-Eye Calibration AX=XB"；论文 "Tsai, R. Y., & Lenz, R. K. (1989). A new technique for fully autonomous and efficient 3D robotics hand/eye calibration."

## 四、相机模型与内参

| 概念 | 说明 |
|------|------|
| **针孔相机模型** | 3D 点通过投影到成像平面：$s p = K [R\|t] P$ |
| **内参矩阵 $K$** | $\begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$ |
| **焦距 $f_x, f_y$** | 像素单位，与传感器像素尺寸相关 |
| **主点 $c_x, c_y$** | 光轴与成像平面的交点 |
| **畸变系数** | 径向畸变 $k_1, k_2, k_3$ + 切向畸变 $p_1, p_2$ |
| **OpenCV 畸变模型** | `cv2.undistort()` / `cv2.initUndistortRectifyMap()` |
| **重投影误差** | 3D 点投影到图像上的像素坐标与检测到的像素坐标之差 |

> **🔗 推荐学习**：OpenCV 官方教程 "Camera Calibration"；《视觉 SLAM 十四讲》第 5 讲。

## 五、ChArUco 标定板

| 概念 | 说明 |
|------|------|
| **ChArUco 板** | 棋盘格 + ArUco 码的混合，兼具两者的优点 |
| **ArUco 码** | 二进制编码的方形标记，可以唯一识别每个 marker |
| **棋盘格角点** | ChArUco 板中间的棋盘交叉点，精度高 |
| **`charuco_board_create()`** | OpenCV 4.x 旧版 API，定义 ChArUco 板 |
| **`interpolateCornersCharuco()`** | 从检测到的 ArUco 码推算棋盘角点 |
| **`estimatePoseCharucoBoard()`** | 估计标定板的位姿（rvec + tvec） |
| **字典 DICT_5X5_100** | 5×5 编码，100 个 marker，项目里用的就是这个 |
| **方格大小 vs. marker 大小** | 项目：30mm 方格，22mm marker |

> **🔗 推荐学习**：OpenCV 官方 "Detection of ChArUco Boards" 教程；ArUco 论文 "S. Garrido-Jurado et al. (2014). Automatic generation and detection of highly reliable fiducial markers under occlusion."

## 六、机器人运动学

| 概念 | 说明 | 代码中的体现 |
|------|------|-------------|
| **正运动学** | 由关节角 → 末端位姿（已知 $q$ 求 $T$） | libfranka `RobotState.O_T_EE` |
| **逆运动学** | 由末端位姿 → 关节角（已知 $T$ 求 $q$） | `panda_py` 内部实现 |
| **末端执行器坐标系** | 法兰盘中心的位姿 | `O_T_EE` 是基坐标系到末端的变换 |
| **$O_T_{EE}$ 矩阵解析** | 16 元素列主序 $\to$ $4\times4$ 矩阵 | `otee_to_transform()` 函数 |
| **工作空间** | 机器人可以到达的区域 | 无碰撞可达范围 |
| **奇异点** | 关节构型导致某个方向自由度丧失 | 运动学逆解不稳定 |

> **🔗 推荐学习**：《机器人学导论》（John J. Craig）；或任何 "Introduction to Robotics" 课程。

## 七、Franka 机器人的控制方式

| 控制方式 | 说明 | 特点 |
|---------|------|------|
| **关节位置控制** | 控制每个关节的目标角度 | 精确的位置追踪，但易产生大关节力矩 |
| **关节速度控制** | 控制每个关节的目标角速度 | 比位置控制更平滑 |
| **笛卡尔位置控制** | 控制末端的位姿（位置+朝向） | 直观，但可能经过奇异点 |
| **笛卡尔阻抗控制** | 控制末端的位置，同时模拟弹簧阻尼系统 | ✅ **本项目用的方法** |
| **关节阻抗控制** | 每个关节模拟弹簧阻尼 | 适合人机交互 |
| **力控制** | 控制末端接触力 | 需要外部传感器或内部力矩估计 |

### 7.1 阻抗控制（重点补）

| 概念 | 说明 |
|------|------|
| **弹簧-质量-阻尼模型** | $M\ddot{x} + D\dot{x} + K(x - x_d) = F_{ext}$ |
| **刚度 $K$** | $3\times3$ 对角矩阵，越大越"硬"，跟踪精度高但碰撞危险 |
| **阻尼比 $\zeta$** | 控制振荡的衰减速度，$\zeta \ge 1$ 为过阻尼，不振荡 |
| **$K=100$ 项目值** | 较低的刚度，让机器人"软"，配合阻尼 1.5 抑制抖动 |
| **`nullspace_stiffness`** | 零空间刚度，控制肘部姿态 |
| **`filter_coeff`** | 指令滤波系数，0~1，越小越平滑 |
| **`CartesianImpedance`** | panda_py 的笛卡尔阻抗控制器 |

> **🔗 推荐学习**：Franka 官方文档 "libfranka control"；论文 "Hogan, N. (1985). Impedance control: An approach to manipulation."

## 八、安全机制

| 概念 | 说明 |
|------|------|
| **`cartesian_reflex`** | Franka 检测到碰撞或异常的外部力矩时自动停止 |
| **`power_limit_violation`** | 功率超出限制，常见于运动过快 |
| **`joint_position_limits_violation`** | 关节超过软限位 |
| **`recover()`** | 清除错误状态（T1 模式下会报错） |
| **`set_default_behavior()`** | 恢复默认控制行为 |
| **T1 示教模式** | 手动引导模式，不能执行自动控制指令 |

> **🔗 推荐学习**：Franka 控制界面（Desk）手册；libfranka `errors.h` 中的所有错误枚举。
